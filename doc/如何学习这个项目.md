# 如何学习这个 RAG 知识库项目

> 适合有一定 Java/Spring 基础，想通过本项目理解 RAG 全栈开发的读者。  
> 项目地址：`org.moyu:rag` · Spring Boot 4 + Spring AI 1.1.7 + pgvector + Vue 3

---

## 学习路线（建议顺序）

### 第一阶段：跑起来（30 分钟）

先别读代码，把项目跑通，建立直观感受。

```
1. 启动 PostgreSQL + Redis
2. 执行 init.sql 建表
3. 复制 application-dev.yml 配置
4. 生成 RSA 密钥
5. 启动后端 → 启动前端
6. 注册 → 登录 → 创建知识库 → 配置模型 → 上传文档 → 对话
```

读完这步，你就知道"这个系统到底长什么样"。

**参考**：`README.md` 的「快速开始」章节

---

### 第二阶段：理解 RAG 核心概念（20 分钟）

RAG 只有三个步骤：

| 英文 | 中文 | 干了什么 | 谁在做 |
|------|------|---------|--------|
| **R**etrieve | 检索 | 把用户问题和知识库中最相似的文档块找出来 | `VectorStoreService` + pgvector |
| **A**ugment | 增强 | 把这些文档块拼到 LLM 的提示词里 | `RagService.systemMsg()` |
| **G**enerate | 生成 | LLM 根据提示词生成答案 | `ModelFactory` → OpenAI API |

关键是理解"嵌入向量"和"余弦距离"：
- 一段文字 → 嵌入模型 → 一串小数（向量）
- 意思相近的文字 → 向量距离也近
- pgvector 的 `<=>` 运算符就是算余弦距离

**参考**：`doc/RAG全流程讲解.md` 第一章

---

### 第三阶段：读文档入库流水线（40 分钟）

这是理解项目最好的入口，因为它涉及面最广。

调用链（建议按这个顺序读代码）：

```
前端上传 KbDetail.vue
  → DocumentController.upload()
    → DocumentServiceImpl.doUpload()
      → 写磁盘（FileUploadUtil）
      → 写数据库（status=0）
      → 异步线程池 → DocumentProcessor.process()
        → Tika 解析文档 → TokenTextSplitter 切块
        → EmbeddingModel 向量化 → VectorStoreService.store()
        → 更新 status=1
```

**关键要理解**：
- 为什么上传完立刻返回 HTTP 200？（异步处理，不阻塞用户）
- 双事务设计的意义？（主事务失败时，`markFailed` 独立事务确保失败状态落库）
- metadata 的 JSONB 字段怎么实现多知识库隔离？（`WHERE metadata->>'kb_id' = ?`）

**参考**：`doc/RAG全流程讲解.md` 第二章、第六章

---

### 第四阶段：读对话查询流水线（30 分钟）

```
前端提问 KbChat.vue
  → chatApi.chatStream() → fetch SSE
    → RagController.chatStream()
      → RagService.retrieve()
        → search() → embed → VectorStoreService.search()
        → 拼 context 字符串
      → buildPrompt() → 系统消息 + 历史消息 + 当前问题
      → ChatModel.stream() → Flux 逐 token 返回
    → SseEmitter 逐 token 推给前端
  → 前端逐 token 更新 UI（ReadableStream + 逐行解析 data: 事件）
```

**先读非流式** `RagService.chat()`（逻辑一样但代码更短），**再读流式** `RagService.chatStream()`。

**参考**：`doc/RAG全流程讲解.md` 第三章、第四章

---

### 第五阶段：理解动态模型工厂（20 分钟）

`ModelFactory` 是项目最有设计感的部分：

```java
// 根据数据库中的 provider 字段，运行时决定用哪个 LLM
return switch(provider) {
    case "openai", "custom" -> openAiChat(config);
    case "anthropic"  -> anthropicChat(config);
};
```

每个知识库可以配不同的 API Key、不同的模型、不同的提供商。  
**理解这点后，你就能回答"为什么 model_config 表里每个配置都有 kb_id"。**

**参考**：`README.md` 的「扩展指南」、`ModelFactory.java`

---

### 第六阶段：安全体系（15 分钟）

```
传输中：RSA 公钥加密（前端加密 → 后端解密）
存储时：AES-256 加密（数据库中 api_key 是密文）
使用中：内存明文（调用 LLM API 时需要明文）
```

RSA 用于**传输**，AES 用于**存储**，JWT 用于**认证**，角色用于**授权**。  
每一层解决不同的问题。

**参考**：`README.md` 的「安全体系」、`AesEncryptor.java`、`RsaUtil.java`

---

### 第七阶段：读前端（30 分钟）

前端比后端简单得多，重点是理解几个模式：

| 模式 | 代码位置 |
|------|---------|
| API 封装 | `src/api/*.ts` — 每个文件对应一个后端 Controller |
| 认证状态 | `src/stores/auth.ts` — Pinia store + localStorage |
| 统一请求 | `src/api/request.ts` — Axios 拦截器（自动带 token、401 跳登录） |
| SSE 流式 | `KbChat.vue` → `sendMessage()` — ReadableStream + 逐行解析 |

**前端不需要从头读到尾**。打开 `KbDetail.vue` 看一个增删改查的例子，再看 `KbChat.vue` 的 SSE 流式处理就够了。

**参考**：`frontend/README.md`

---

### 第八阶段：扩展能力（按需阅读）

如果你想给项目加功能，这些是入口：

| 想做什么 | 需要改什么 |
|----------|-----------|
| 接入新 LLM 提供商 | `ModelFactory` 加 switch 分支 + pom.xml 加依赖 |
| 加新的角色类型 | `KbRole` 枚举加值，`atLeast()` 自动生效 |
| 自定义限流 | `@RateLimit` 注解的参数 |
| 支持新文件类型 | `FileUploadUtil.DOCUMENT_EXTENSIONS` 加扩展名 |
| 加聊天历史导出 | 前端调 `GET /chat/history` + 后端加导出接口 |

**参考**：`README.md` 的「扩展指南」

## 关键点自测

读完项目后，你应该能回答：

- ❓ 为什么文档处理完才 `status=1`？如果在处理过程中用户搜到了部分 chunk 怎么办？
- ❓ `ModelFactory.createChatModel()` 和 `createStreamingChatModel()` 有什么区别？为什么返回同一个类型？
- ❓ JWT Token 过期了怎么办？前端怎么处理的？
- ❓ 怎么给项目加一个 Ollama 的嵌入模型支持？
- ❓ 前端 SSE 如果断网了怎么恢复？
